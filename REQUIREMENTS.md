# Obsidian 无序列表代码块缩进插件需求文档

## 1. 项目目标

开发一个面向程序员的 Obsidian 插件，解决在无序列表层级中插入代码块时，代码块无法自然缩进到当前列表层级的问题。

插件的唯一功能是：当用户在无序列表的空列表项中插入 fenced code block 时，自动移除空 bullet，并让代码块按照当前列表层级缩进显示。

## 2. 目标用户

主要用户是使用 Obsidian 编写技术笔记、编程教程、源码分析、框架学习笔记的程序员。

典型场景：

- 在无序列表中记录知识点
- 某个知识点下面需要插入代码示例
- 希望代码块视觉上属于当前列表层级
- 不希望手动调整 Markdown 缩进

## 3. 核心需求

### 3.1 支持范围

插件只处理以下场景：

- 当前光标位于无序列表的空列表项中
- 用户插入的是 fenced code block
- fenced code block 使用 Markdown 三反引号格式
- 无序列表符号第一版只支持 `-`、`*`、`+`

示例 fenced code block：

````md
```ts
code
```
````

### 3.2 不支持范围

插件第一版不处理：

- 裸代码自动识别
- 自动给裸代码包裹代码块
- 有序列表
- 任务列表 `- [ ]`
- 引用块里的列表
- 表格里的代码块
- HTML 代码块
- 自动识别语言类型
- 快捷键插入代码块
- 自定义 UI 或设置页

## 4. 期望行为

### 4.1 粘贴完整代码块

用户当前内容：

```md
- 触发缓存的四个条件
  - GET方法
  - |
```

用户粘贴：

````md
```ts
export async function GET() {
  return Response.json({ data: "hello" });
}
```
````

插件应自动变成：

````md
- 触发缓存的四个条件
  - GET方法

  ```ts
  export async function GET() {
    return Response.json({ data: "hello" });
  }
  ```
````

### 4.2 手动输入代码块开头

用户当前内容：

```md
- 触发缓存的四个条件
  - GET方法
  - |
```

用户输入：

````md
```ts
````

插件应立即把当前空 bullet 转换为缩进代码块起始位置：

````md
- 触发缓存的四个条件
  - GET方法

  ```ts
````

用户后续继续手动输入代码内容和 closing fence。

## 5. 缩进规则

插件根据当前空无序列表项的前导空格计算缩进。

例如当前行是：

```md
  - |
```

前导缩进是两个空格，插件移除 `- ` 后，代码块使用同样的两个空格缩进：

````md
  ```ts
  code
  ```
````

如果当前行是：

```md
    - |
```

则生成：

````md
    ```ts
    code
    ```
````

## 6. Markdown 语义说明

本插件采用“方案 B”。

也就是：代码块不会被保留为一个带 bullet 的列表项内容，而是删除当前空 bullet，使代码块在视觉上对齐到当前列表层级。

需要接受的限制是：Markdown 本身没有“无 bullet 的列表项”。因此该方案更准确地说是：

**让代码块缩进到用户当前所在的无序列表层级。**

而不是严格创建一个“和上一个 bullet 同级但没有 bullet 的列表项”。

## 7. 触发条件

插件只在满足以下条件时生效：

1. 当前编辑器处于 Markdown 编辑状态
2. 当前光标所在行是空无序列表项
3. 当前行格式匹配：

```regex
^(\s*)[-*+]\s*$
```

4. 用户插入内容以 fenced code block 开始：

````md
```
````

## 8. 非触发示例

以下情况插件不应修改内容。

光标不在空 bullet 上，不处理：

```md
- GET方法
```

当前列表项已有内容，不处理：

```md
- GET方法
  - 普通文字
```

裸代码，不处理：

```md
- GET方法
  - export async function GET() {}
```

有序列表不是第一版目标：

```md
1. GET方法
   - |
```

## 9. 插件形态

第一版可以没有设置页、没有命令、没有 ribbon 图标。

插件加载后自动监听编辑器输入或粘贴事件，在满足条件时执行格式修正。

## 10. 验收标准

插件完成后，应满足：

- 在无序列表空 bullet 上粘贴 fenced code block，会自动删除 bullet 并保留缩进
- 在无序列表空 bullet 上手动输入 ```` ``` ````，会自动删除 bullet 并保留缩进
- 不会修改普通文本、普通列表项、裸代码
- 不会影响 Obsidian 原有列表缩进、换行、Tab 行为
- 插件卸载后不残留事件监听器
- `npm run build` 能成功生成 `main.js`
